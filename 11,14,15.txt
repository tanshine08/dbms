For 15th practical (Restore Database), you FIRST need a backup file.
Without backup file, restore cannot happen.

So actual flow is:

REAL FLOW OF RESTORE PRACTICAL

Create Database
→ Create Table
→ Insert Records
→ Take Backup
→ Drop Database
→ Restore Database
→ Check Restored Data


That is the proper logic.

SIMPLE COMPLETE FLOW OF 15
STEP 1 — Open CMD

cmd

STEP 2 — Go to MySQL bin Folder

cd C:\Program Files\MySQL\MySQL Server 8.0\bin

STEP 3 — Open MySQL

mysql -u system -p


Password:


root

STEP 4 — Create Database

CREATE DATABASE collegeDB;

STEP 5 — Use Database

USE collegeDB;

STEP 6 — Create Table

CREATE TABLE Student(
    StudentID INT PRIMARY KEY,
    Name VARCHAR(50)
);

STEP 7 — Insert Records

INSERT INTO Student VALUES
(1,'Tanmay'),
(2,'Rahul');

STEP 8 — Display Table

SELECT * FROM Student;


You will see records.

STEP 9 — Exit MySQL

EXIT;

STEP 10 — Take Backup

mysqldump -u system -p collegeDB > college_backup.sql


Password:


root


Now backup file is created.

STEP 11 — Open MySQL Again

mysql -u system -p


Password:


root

STEP 12 — Drop Database

DROP DATABASE collegeDB;


Now database deleted.

STEP 13 — Create Empty Database Again

CREATE DATABASE collegeDB;

STEP 14 — Exit MySQL

EXIT;

STEP 15 — Restore Database

mysql -u system -p collegeDB < college_backup.sql


Password:


root


Now backup restored.

STEP 16 — Open MySQL Again

mysql -u system -p


Password:


root

STEP 17 — Use Database

USE collegeDB;

STEP 18 — Check Restored Table

SELECT * FROM Student;


You will again see:


1  Tanmay
2  Rahul


That proves restore successful.

WHY DROP DATABASE?

External may ask this.

Answer:

“To check whether backup can recover deleted database data.”

Very good viva answer.

Viva Questions and Answers
Q1. What is mysqldump?

Answer:
mysqldump is a utility used to take backup of MySQL database.

Q2. Where is mysqldump executed?

Answer:
In CMD or terminal, not inside MySQL.

Q3. What is created after backup?

Answer:
A .sql backup file is created.

Q4. How do you know backup succeeded?

Answer:
Backup .sql file gets created in current folder.
