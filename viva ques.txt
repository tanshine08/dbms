BASIC DBMS QUESTIONS
1. What is DBMS?

DBMS stands for Database Management System. It is used to store, manage and retrieve data.

2. What is database?

A database is an organized collection of data.

3. What is table?

A table stores data in rows and columns.

4. What is row?

A row represents one record.

5. What is column?

A column represents one attribute or field.

KEY QUESTIONS
6. What is Primary Key?

Primary key uniquely identifies each record in a table.

Example:
StudentID.

7. Properties of Primary Key
Unique
Cannot be NULL
8. What is Foreign Key?

Foreign key creates relationship between two tables.

9. Why use Foreign Key?

To maintain referential integrity.

10. Difference between Primary Key and Foreign Key
Primary Key	Foreign Key
Unique	Can repeat
Cannot be NULL	Can be NULL
Identifies record	Connects tables

AGGREGATE FUNCTION QUESTIONS
21. What are aggregate functions?

Functions that perform calculations on multiple rows.

Examples:

SUM()
AVG()
COUNT()
MAX()
MIN()
22. What does SUM() do?

Calculates total.

23. What does COUNT(*) do?

Counts total rows.

24. What does MAX() do?

Returns highest value.

25. What does MIN() do?

Returns smallest value.

GROUP BY QUESTIONS
26. What is GROUP BY?

GROUP BY groups similar rows together.

27. Why use GROUP BY?

To apply aggregate functions on grouped data.

28. Difference between WHERE and HAVING
WHERE	HAVING
Filters rows	Filters groups
Used before GROUP BY	Used after GROUP BY
JOIN QUESTIONS
29. What is JOIN?

JOIN combines records from multiple tables.

30. Types of JOIN
INNER JOIN
LEFT JOIN
RIGHT JOIN
FULL JOIN
31. What is INNER JOIN?

Returns matching records from both tables.

32. What is LEFT JOIN?

Returns all records from left table and matching records from right table.

33. Difference between INNER JOIN and LEFT JOIN
INNER JOIN	LEFT JOIN
Only matching rows	All left table rows

BACKUP & RESTORE QUESTIONS
53. What is Backup?

Copy of database stored for recovery.

54. Why do we take Backup?

To recover lost or corrupted data.

55. What is mysqldump?

Utility used to take MySQL backup.

56. Where is mysqldump executed?

In CMD or terminal.

57. What file is created after backup?

.sql file.

58. What is Restore?

Recovering database from backup file.

59. Command for Backup
mysqldump -u system -p collegeDB > college_backup.sql
60. Command for Restore
mysql -u system -p collegeDB < college_backup.sql
61. How to verify backup success?

Check .sql file using:

dir
62. How to verify restore success?
SELECT * FROM Student;
